# Glossary

**Execution** — The act of changing system state irreversibly.

**Runtime** — The period after execution has begun.

**Constraint** — A pre-committed condition that must be satisfied for execution.

**Admissible** — Allowed to execute under defined constraints.

**512** — A minimal execution admissibility kernel.

**CVS** — A disjoint system that records execution evidence.

**Fail-fast** — Refusal to execute invalid actions.

**Authority** — The ability to permit or deny execution before it occurs.

**Latency** — Time between decision and action.

**Machine-speed** — Faster than human intervention.
