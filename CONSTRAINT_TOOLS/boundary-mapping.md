# Execution Boundary Mapping

Execution Boundary Mapping is the process of identifying where actions become irreversible.

---

## Purpose

To distinguish:
- planning vs execution
- intent vs action
- reversible vs irreversible state change

Only execution requires constraint.

---

## Method

1. Identify actions that change system state
2. Locate the point of irreversibility
3. Determine whether human intervention can still arrive in time
4. Classify the action as:
   - constraint-eligible
   - downstream-only

---

## Output

- a map of execution surfaces
- identified constraint candidates
- excluded workflows

---

## Final Note

If humans cannot intervene before completion, the action must be constrained or removed.
